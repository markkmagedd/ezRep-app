# Implementation Plan: Exercise Selector Screen Redesign

**Branch**: `007-redesign-exercise-selector` | **Date**: 2026-03-13 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/007-redesign-exercise-selector/spec.md`

## Summary

Completely restructure the layout of `ExerciseSelectorScreen.tsx` to eliminate the layout bug where the selection pile and absolute-positioned Add button overlap the exercise browse list. Replace with a 3-zone flex layout: fixed header (search + filters), `flex: 1` browse list, and an animated sticky bottom tray that slides up when selections are made.

## Phase 0: Research Summary

See [`research.md`](./research.md) for full decisions. Key outcomes:

1. **Layout**: Sticky bottom tray as a normal-flow `View` sibling to the `FlatList` — guaranteed non-overlap by the flex engine.
2. **Animation**: `Animated.Value` driving `translateY`; no new dependencies.
3. **Many selections**: Horizontal `FlatList` in fixed-height tray; count shown on button.
4. **State**: `selectionBuffer` local state unchanged; `Animated.Value` in a `useRef`.

## Phase 1: Design

### Layout Architecture

```
<SafeAreaView flex:1>
  <View>                          ← Search bar (fixed)
  <View>                          ← Body-part chips (fixed)
  <View>                          ← Equipment chips (conditional, fixed)
  <FlatList flex:1 />             ← Browse list — fills remaining space
  <Animated.View translateY>      ← Bottom tray (slides in/out)
    <FlatList horizontal />       ← Selected chips
    <TouchableOpacity />          ← Add button
  </Animated.View>
</SafeAreaView>
```

### Animation Specification

```
TRAY_HEIGHT = 120  (chip row ~44px + button ~52px + padding ~24px)

Entry:  selectionBuffer.length 0→1
        Animated.timing(trayAnim, { toValue: 0, duration: 200, easing: Easing.out(Easing.ease) })

Exit:   selectionBuffer.length 1→0
        Animated.timing(trayAnim, { toValue: TRAY_HEIGHT, duration: 200, easing: Easing.in(Easing.ease) })
```
